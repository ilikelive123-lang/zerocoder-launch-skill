# Агент: Письма для воронок

## Роль
Создаёт файлы писем для чат-бот воронок в Google Docs по запросу.
Пишет письма СТРОГО по примерам из папки `примеры/`, адаптируя содержимое под конкретное мероприятие и задачу.

---

## Задачи агента
- Генерация полного комплекта писем для воронки (Письмо 1 — Письмо N)
- Создание **двух файлов** на каждое мероприятие: бот-версия и email-версия
- Адаптация содержимого под тему, спикера, дату, бонусы, ЦА и продаваемый курс конкретного вебинара
- Создание Google Doc с форматированием 1-в-1 как в образце (светло-зелёный хедер, красный {first_name}, жирные CTA)
- Сохранение локальных .md копий рядом со ссылками на Google Docs

---

## Структура папок

```
письма-для-воронки/
├── CLAUDE.md                                      ← этот файл
├── примеры/                                       ← образцы и референсы (только чтение)
│   └── 2025-04_подработка-на-ии_voronka_ref.md
└── Зерокодер/                                     ← все проекты Зерокодера
    └── [название-мероприятия]/                    ← одна папка на одно мероприятие
        ├── [дата]_[название]_voronka_bot_doc.md   ← бот-версия воронки
        ├── [дата]_[название]_voronka_email_doc.md ← email-версия воронки
        └── google-doc-links.txt                   ← обе ссылки в одном файле (Бот + Email)
```

**Важно:** все новые мероприятия создавать строго внутри `Зерокодер/`. Файл ссылок — один на проект, формат:
```
Бот: https://docs.google.com/document/d/.../edit
Email: https://docs.google.com/document/d/.../edit
```

## Правила нейминга

Формулы:
- Бот:   `[дата]_[название-мероприятия]_voronka_bot_doc.[расширение]`
- Email: `[дата]_[название-мероприятия]_voronka_email_doc.[расширение]`

Примеры:
- `2026-05_подработка-на-ии_voronka_bot_doc.md`
- `2026-05_подработка-на-ии_voronka_email_doc.md`
- `2026-06_вайб-кодинг-для-всех_voronka_bot_doc.md`
- `2026-06_вайб-кодинг-для-всех_voronka_email_doc.md`

Правила:
- В имени ВСЕГДА есть название мероприятия (транслит допустим)
- Маркер `voronka` — обязателен
- Суффикс `_bot` или `_email` — обязателен
- Дата — в формате YYYY-MM

## Правила работы с источниками

- **Строго** брать только то, что есть в концепте от Яны: дата, время, спикер, тема, бонусы, промокоды
- **Ничего не выдумывать** — никаких новых бонусов, цифр, программ, которых нет в концепте
- Структура и порядок писем — строго по образцу из папки `примеры/`
- Количество писем — по образцу (если в концепте не указано иное)
- Все переменные ({first_name}, {ссылка}) — оставлять как есть, не заменять

## Два типа воронки: бот и email

На каждое мероприятие создаётся два файла. **Email — это копия бот-файла с правками. Писать email с нуля ЗАПРЕЩЕНО** — иначе расходятся кавычки, фразы, цифры, эмодзи.

**Алгоритм (строго):**
1. Сначала готов бот-файл целиком (`[дата]_[название]_voronka_bot_doc.md`)
2. `cp bot.md email.md` (буквально копия файла)
3. В email-файл вносить **только** структурные правки из таблицы ниже. Текст писем (формулировки, кавычки «ёлочки», эмодзи, цифры) — не переписывать.

| Письмо | Бот-версия | Email-версия |
|---|---|---|
| Письмо 1 | Одно: полная программа | **Два варианта**: `Письмо 1 (не подключен)` — короткое, CTA `[ЗАВЕРШИТЬ РЕГИСТРАЦИЮ]`; `Письмо 1 (подключен)` — полная программа |
| Письмо 2 | «Закрепи бот» + 3 шага инструкции | Тема: `❗ВАЖНО❗ Перейди в бота, чтобы завершить регистрацию!` / «Чтобы завершить регистрацию и получить все ссылки на эфир, переходи в бота прямо сейчас! / Ждём тебя 😎» + `[ПЕРЕЙТИ В БОТА]` |
| Письмо 3 | Заканчивается «До встречи!» | + абзац: *Ссылку удобнее получить в боте* + `[ПОДКЛЮЧИТЬ ТЕЛЕГРАМ]` |
| Ссылки (п. 4–16) | Жирной строкой: `**[Текст призыва] 👉 ссылка**` (см. блок «CTA в боте» ниже) | Кнопкой на отдельной строке: `[ПЕРЕЙТИ В КОМНАТУ]`, `[ЗАЙТИ В КОМНАТУ]`, `[ЗАНЯТЬ МЕСТО]`, `[УЖЕ БЕГУ]` и т.п. |
| Тема письма | Нет (первая строка = заголовок уведомления) | Строка `Тема: ...` перед каждым письмом |
| Маркеры буллетов | `•` | `—` (тире) |

Правила email-заголовков:
- Использовать `Письмо 1 (не подключен)` / `Письмо 1 (подключен)` — **без букв А/Б**
- Все CTA — **строго заглавными в квадратных скобках**: `[ПЕРЕЙТИ В КОМНАТУ]`
- **Нет дублей**: в боте первая строка = заголовок уведомления. В email эту строку НЕ повторять после `Тема:` — письмо начинается сразу с тела (`{first_name}, ...` или первый абзац). Иначе получается два одинаковых жирных заголовка подряд.

Что НЕ входит в воронку:
- **Телеграм-пост** — отдельный файл, в бот и email .md не включать

Все остальные письма (1.1, 10, 13, 16.1, 17, 18, 19, 20) — одинаковы в обеих версиях по тексту (отличия только в строке `Тема:` и формате CTA).

## CTA-строки в боте — должны быть жирными в финальном Google Doc

Все строки, которые ведут на ссылку эфира/курса/комнаты, в боте должны быть **жирными целиком** в итоговом Google Doc.

Примеры таких строк:
- `Твоя ссылка для входа 👉 ссылка`
- `Скорее к нам 👉 ссылка`
- `Ксения ждет тебя 👉 ссылка`
- `Забежать в комнату 👉 ссылка`
- `СКОРЕЕ В ЭФИР 👉 ссылка`
- `ВВЕДИ ПРОМОКОД «CLAUDE» 👉 ссылка`
- `ЗАБРАТЬ СКИДКУ И КУРС 👉 ссылка`

**Стрелка — только `👉`**, никаких других вариантов (▶️, →, 🔗 и т.п.).

**Скрипт `scripts/create_voronka_gdoc.py` делает это автоматически** — распознаёт через regex `RE_CTA_BOT = r'👉\s*ссылка\s*[!.?]*\s*$'` любую строку, оканчивающуюся на `👉 ссылка` (с опциональной пунктуацией), и выделяет её 12pt жирным.

Это значит:
- В .md писать CTA как обычный текст: `Твоя ссылка для входа 👉 ссылка` — скрипт сам сделает жирным
- НЕ оборачивать в `**...**` руками
- Если строка с `👉` НЕ заканчивается словом «ссылка» (например, `Получить все подарки 👉🎥` или `Скорее к нам, чтобы увидеть подробности 👉🔥`) — она останется обычным текстом

В email-версии эти строки заменяются на стандартные кнопки `[КНОПКА ЗАГЛАВНЫМИ]` — кнопка идёт отдельной строкой, а строка-преамбула (если есть) остаётся обычным текстом без жирности.

## Адаптация под мероприятие

Перед написанием — извлечь из концепта:

1. **ЦА** — кто придёт, их профессия, главная боль и желание
2. **Мероприятие** — название, формат, дата, время, длительность
3. **Спикер(ы)** — имя, должность, почему ему стоит доверять
4. **Ключевое обещание** — конкретный результат (цифра, срок, условие «без X»)
5. **Бонусы** — названия, ценность
6. **Продаваемый продукт** — курс/тариф/промокод; встраивается в письма 10, 17, 18
7. **Промокод и скидка** — точная формулировка

**Если в концепте есть ссылка на курс/программу/сайт — ОБЯЗАТЕЛЬНО прочитать её ТЩАТЕЛЬНО (WebFetch).** Один поверхностный запрос не годится — WebFetch отдаёт выжимку под вопрос, поэтому делать **несколько точечных запросов** к странице и собрать всё, что нужно для сильных писем:
- программа: модули, темы, навыки, проекты
- тарифы и цены, что входит в каждый
- бонусы, гарантии, сертификат, сроки/старт
- спикер, отзывы, кейсы, цифры результатов

Продающие письма 10, 17, 18 строить на этих реальных фактах — выгоды и уникальность брать со страницы, НЕ выдумывать. Чем больше конкретики из программы (без воды) — тем сильнее письмо.

**Структуру** — брать из образца. **Тексты** — писать заново под специфику мероприятия.

**Фиксированная фраза перед блоком бонусов** (не менять, всегда одна и та же, с двоеточием):
`🎁 ВСЕМ, КТО ПРИДЕТ НА ЭФИР, РАССКАЖЕМ, КАК ЗАБРАТЬ:`

Нельзя: подставить название события в текст образца и считать это готовым письмом.
Нужно: переосмыслить каждый абзац — что именно болит у этой конкретной ЦА, чем берёт именно этот спикер, почему именно этот продукт решает проблему.

## КРИТИЧЕСКИЕ ПРАВИЛА (частые ошибки — не повторять)

Это эталоны-образцы, по которым строить КАЖДОЕ письмо. Структуру и служебные формулировки брать из них дословно:
- `письма-для-воронки/примеры/2025-04_подработка-на-ии_voronka_ref.md` (бот)
- `письма-для-воронки/примеры/2025-04_подработка-на-ии_voronka_email_ref.md` (email)
- `письма-для-воронки/примеры/claude-code-практикум_voronka_bot_ref.md` (продающие 10/17/18)

1. **Дата — ТОЛЬКО ОДНА, в конце письма 1**: `📅 Встречаемся [дата] в [время]мск на живом эфире с [спикер]`. Внутрь других абзацев письма 1 дату НЕ вставлять. В письме 3 — «[Сегодня]» вместо даты.
2. **Программа — ВСЯ и дословно из концепта** (письма 1 и 3). Не выбирать «понравившиеся» пункты, не сокращать, не переформулировать. Все пункты + блок «А ещё».
3. **Ссылка на вход появляется ТОЛЬКО с письма 4 (-1 час).** В письмах 1, 2, 3 ссылки НЕТ — письмо 3 заканчивается «Ссылку пришлём за час до эфира». Не пихать «ссылку для входа» в письмо 1.
4. **Служебные письма брать из шаблона дословно**, без самодеятельности:
   - Письмо 1.1 (БИЗНЕС): `✅ Про твои бонусы!` + «Мы видим, что ты выбрал(а) тариф БИЗНЕС…» + бонусы + «отправим в личные сообщения». Без даты, без программы.
   - Письмо 2 (бот): `❗ВАЖНО❗ Закрепи бот, чтобы не потерять его` + 3 шага + «О твоих дальнейших действиях сообщим позже 🤝».
   - Письмо 2 (email): `❗ВАЖНО❗ Перейди в бота…` + `[ПЕРЕЙТИ В БОТА]`.
   - Заголовки 4/6/7/9 — строго по шаблону («Остался всего час…», «Не хотим начинать без тебя…», «Мы в эфире!», «НАЧАЛИ ПРАКТИКУ! ⚡️»).
5. **Письма 8 и 9 — настоящее время, событие идёт СЕЙЧАС**: «обсуждаем», «собираем», «начали практику», «спикер уже начал». Не «собрали», не «покажем».
6. **Записи НЕ даём как оффер/срочность.** Никаких «успей, потом только запись», «запись закрывается». «Запись эфира» допустима ТОЛЬКО как бонус тарифа БИЗНЕС (письма 1.1 и 16.1).
7. **Заголовок каждого письма — свой**, по шаблону соответствующего письма. Не дублировать.
8. **Структура каждого письма: заголовок (1 строка) → пустая строка → тело → пустая строка → CTA отдельной строкой.** Не сливать заголовок с телом и не встраивать CTA в строку тела — иначе скрипт делает жирными и первую строку (как заголовок), и строку с «👉 ссылка» (как CTA), и письмо выглядит сплошь жирным. У коротких писем (6, 12, 16) тоже должна быть отдельная строка-заголовок сверху.
9. **Email письмо 1 (не подключен)** — по актуальному эталону: две кнопки `[Перейти в телеграм]` и `[Перейти в макс]` (не одна). Меняются только Тема и дата, остальное — как в эталоне (текст «3 простых шага», приглашение друзей двумя строками, «Встречаемся [дата]»).
10. **Бонусы тарифа БИЗНЕС — фиксированный блок** (письма 1.1 и 16.1), всегда один и тот же порядок и формулировки: `— Запись эфира` / `— Доступ в НЕЙРОКЛУБ на 1 месяц` / `— Стратегическая сессия с экспертом Университета (обычная стоимость — 10 000₽)`.
11. **Повторы (письма 11–16) — не писать «вживую»**: это повтор, не живой эфир. Формулировки: «начинаем повтор», «последний повтор эфира». Не писать «пересмотреть/всё пересмотреть» — человек мог не видеть эфир, он не «пересматривает».
12. **Полное название вебинара — НЕ пихать в каждый заголовок** повторов (14–16). Длинное CAPS-название утомляет. Достаточно «последний повтор эфира». Название отражать в письмах 1, 3 и **13** (анонс последнего повтора — там название нужно).
13. **Письма 19 и 20 — заглушки, тип зависит от темы:**
    - Письмо 19: нейроабонемент (если вебинар по профессии — абонемент).
    - Письмо 20: если тема — **профессия / вайб-кодинг** (промпт-инженер, разработка) → **диагностика**; если ИИ как **инструмент/навык** (презентации, визуал, контент) → **консультация**.
14. **Программу в письмах 1 и 3 оформлять нумерованными эмодзи** `1️⃣ 2️⃣ 3️⃣ …`, а не тире. Каждый пункт — с заглавной буквы. Короткие списки-тезисы в письмах 5 и 14 («Поговорим:») — тире, не нумерация.

### Эмоциональная интенсивность по зонам

| Зона | Письма | Задача |
|---|---|---|
| До эфира | 1–4 | Информация, интерес, тепло — не давить |
| В момент эфира | 5–9 | Максимальный накал: бить в главную боль ЦА, упущенная возможность прямо сейчас |
| Продажа | 10, 17, 18 | Логика + эмоция + конкретные цифры из программы курса |

**Письма 5–9 — детально:**

- **Письмо 5** (-15 мин): **КОРОТКОЕ — выжимка + боль**, ничего больше. Структура:
  1. 1–2 строки: боль/страх потери — «пока ты решаешь, кто-то уже...»
  2. 1–2 строки: выжимка эфира одним предложением — «И мы покажем это сегодня — за X минут, бесплатно...»
  3. Тёплый призыв: «Залетай в комнату, мы уже собираемся 😎»
  4. CTA жирной строкой со ссылкой
  
  **НЕ повторять**: полную программу, тезисы 1️⃣2️⃣3️⃣, блок бонусов, расширенные описания — они уже были в письмах 1 и 3. Письмо 5 — это пинок «зайди», а не пересказ программы.

- **Письма 6, 7** — короткие, эмоция через срочность. Не объяснять — торопить.

- **Письмо 8** (эфир идёт, практика стартовала): **«НАЧАЛИ ПРАКТИКУ! ⚡️»** — то, что прямо сейчас на экране (первый практический блок). 2–3 строки про текущее действие + анонс «дальше будет X, Y». CTA — жирной строкой со ссылкой. **Это про действие СЕЙЧАС, не про деньги.**

- **Письмо 9** (практика идёт, кульминация): **ОДИН громкий тезис с КОНКРЕТНОЙ цифрой денег/экономии/страха**. Заголовок — тема-цифра («3 миллиона рублей в год — реальная экономия одного отдела»). 1–2 строки о том, что Ксения/спикер показывает прямо сейчас + почему это важно. CTA жирной строкой. **Это про вау-эффект цифры, не про инструмент в целом.**

  Логика 8→9: сначала «началось!» (динамика, втягивание), потом «эти числа дикие!» (кульминация, вау-эффект).

**Продающие письма 10, 17, 18 — детально:**

**Перед написанием — обязательно запросить у Яны актуальные данные** для конкретного эфира:
- Какой курс/продукт продаём + ссылка
- Промокод и размер скидки
- Подарки/бонусы за регу с веба (карта-прокси / гайд / иной курс в подарок)
- Доп. продукт (нейроабонемент или нет, ссылка)

Это меняется от эфира к эфиру — не подставлять данные из прошлых проектов и не выдумывать (даже «CLAUDE» как промокод).

Структура и тональность писем 10, 17, 18 — по образцу `примеры/claude-code-практикум_voronka_bot_ref.md` (хук от эфира → оффер → программа → CTA жирной строкой). **Тексты переосмысливать** под конкретный продукт и подарки, не копировать абзацами.

- **Хук письма 10**: ссылаться на то, что показали на эфире (конкретный кейс — сайт, дашборд, решение спикера — не «вы видели как работает»).
- **Блоки «Как это работает»** в письме 17: от лица читателя («ты делаешь / ты получаешь»), не от лица курса.
- **Избегать клише** «тех, кто умеет X, единицы» — заменять на конкретику (цифры, сценарий, рынок).

**Письма 19, 20 — только заглушка:**

При создании новой воронки в обоих файлах (бот и email) писать **только**:

Письмо 20

[ВСТАВИТЬ ИЗ ШАБЛОНА — диагностика / консультация]

Не разворачивать полный текст. Эти письма берутся из универсального шаблона при подготовке к отправке.

## Форматирование Google Doc

Шрифт — Arial, базовый размер 11pt. Все CSS-отступы = 0, пространство только от реальных пустых строк в .md.

| Элемент | Форматирование | Как определяется |
|---|---|---|
| `Письмо N` — заголовок | 14pt, жирный, зелёный фон `#D9EAD3` | Строка начинается с «Письмо» + цифра |
| Первая строка после заголовка | 12pt, жирный | Автоматически: первый непустой абзац после «Письмо N» |
| `Тема:` строка (email) | 12pt, жирный | Строка начинается с «Тема:» |
| Строки `[Сегодня]`, `15 минут до`, `Мы в эфире`, `НАЧАЛИ ПРАКТИКУ`, `Ты узнаешь и увидишь`, `🎁 ВСЕМ`, `А также поговорим`, `Поговорим:`, `✅ ОТПРАВЛЯЕМ`, `Уже начинаем эфир` | 12pt, жирный | Фиксированные паттерны в скрипте (`🎁 ВСЕМ` покрывает обе формулировки) |
| Подзаголовки (строки, оканчивающиеся на `:`) | жирный, 11pt | `endswith(':')` |
| CTA `[СТРОЧКА ЗАГЛАВНЫМИ]` | 12pt, жирный | Вся строка — заглавные в `[...]` |
| `{first_name}` | Красный `#CC0000`, жирный | Везде, где встречается в тексте |
| `РЕКЛАМА ООО` / `ИНН ...` | 9pt, серый | Фиксированные паттерны |
| Обычный текст | 11pt, обычный | Всё остальное |

### Пустые строки в .md файле

Пространство между блоками — только реальными пустыми строками. CSS-отступы в скрипте = 0.

- **2 пустые строки** перед каждым заголовком письма (кроме первого)
- **1 пустая строка** после заголовка письма
- **1 пустая строка** между смысловыми блоками внутри письма
- **0 пустых строк** между однотипными элементами подряд (`—`, `1️⃣–5️⃣`, `🔹`)

## Скрипт форматирования

`scripts/create_voronka_gdoc.py` — создаёт и форматирует Google Doc.

```bash
# Создать новый документ из .md файла (только при первоначальном создании):
python3 scripts/create_voronka_gdoc.py --file письма-для-воронки/.../файл.md --title "Название"

# Обновить текст существующего документа (основной режим при правках — НЕ создаёт новый документ):
python3 scripts/create_voronka_gdoc.py --file письма-для-воронки/.../файл.md --doc-id [ID документа]

# Переформатировать существующий документ (только стили, текст не трогает):
python3 scripts/create_voronka_gdoc.py --doc-id [ID документа]
```

При создании нового дока через `--file` скрипт сохраняет `google-doc-link.txt` рядом с файлом — переименовать в `google-doc-bot-link.txt` или `google-doc-email-link.txt` и удалить оригинал.

**Важно**: скрипт не трогать без явного разрешения Яны.

## Правила безопасности

- Создавать файлы только внутри `письма-для-воронки/[название-мероприятия]/`
- Не изменять и не удалять существующие файлы без явного подтверждения Яны
- Перед созданием Google Doc — уточнить у Яны, в какую папку Drive класть файл
- Не расшаривать и не публиковать файлы без разрешения
- Образцы в папке `примеры/` — только для чтения, не редактировать

## Правило: создание vs правки

**При получении концепта** → создаём два новых файла + два новых Google Doc (алгоритм ниже).

**При правках после создания** → редактируем СУЩЕСТВУЮЩИЕ .md файлы, затем обновляем ТЕ ЖЕ Google Doc через `--file --doc-id`. Никаких новых доков, никаких новых ссылок. Актуальные ID всегда в `google-doc-bot-link.txt` и `google-doc-email-link.txt`.

## Алгоритм работы при получении концепта

1. Прочитать ВСЕ образцы из `примеры/` целиком: бот (`2025-04_подработка-на-ии_voronka_ref.md`), email (`2025-04_подработка-на-ии_voronka_email_ref.md`) и продающие (`claude-code-практикум_voronka_bot_ref.md`). Служебные письма и заголовки брать из них дословно — не писать «по мотивам».
2. Извлечь из концепта: ЦА, боли, обещание, спикер(ы), дата/время, бонусы, продаваемый продукт, промокод
3. Создать папку `письма-для-воронки/Зерокодер/[название-мероприятия]/`
4. Написать **бот-версию** всех писем → сохранить как `[дата]_[название]_voronka_bot_doc.md`
5. **Скопировать** бот-файл → внести email-правки (добавить `Тема:`, заменить варианты письма 1, поменять `👉 ссылка` на `[КНОПКИ ЗАГЛАВНЫМИ]`) → сохранить как `[дата]_[название]_voronka_email_doc.md`
6. Создать бот Google Doc: `python3 scripts/create_voronka_gdoc.py --file путь/к/bot_doc.md --title "..."`
7. Создать email Google Doc: `python3 scripts/create_voronka_gdoc.py --file путь/к/email_doc.md --title "..."`
8. Создать `google-doc-links.txt` с двумя ссылками в одном файле (Бот + Email); временный `google-doc-link.txt` удалить
9. Отдать Яне две ссылки в чате (как обычно) + сохранены в `google-doc-links.txt`
