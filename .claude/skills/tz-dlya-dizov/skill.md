# Агент: ТЗ на лендинг для дизайнера

## Роль
Получает концепт мероприятия от Яны → создаёт готовое ТЗ на лендинг для дизайнера сразу в Google Doc с форматированием по образцу.

## Задачи агента
- Сборка полного ТЗ по структуре блоков из примеров
- Адаптация дизайн-строки под тему, стиль и референс из концепта
- Подстановка спикера по правилам (Зайцева / Пшинник / другой)
- Самостоятельное составление глоссария из всех иностранных слов и сервисов в тексте
- Создание Google Doc с форматированием 1-в-1 как в образце
- Сохранение ссылки в `google-doc-link.txt` рядом с файлом

## Правило: создание vs правки

**При получении нового концепта** → создать новый Google Doc, сохранить ID в `google-doc-link.txt`.

**При любых правках после создания** → редактировать СУЩЕСТВУЮЩИЙ документ через `--file --doc-id`. Никаких новых доков, никаких новых ссылок. ID всегда брать из `google-doc-link.txt` в папке мероприятия.

```bash
# Обновить существующий документ:
python3 scripts/create_landing_tz_gdoc.py --file путь/к/файлу.md --doc-id [ID из google-doc-link.txt]
```

## Глоссарий

Всегда составлять самостоятельно из текста ТЗ. Включать:
- Все англоязычные названия сервисов и инструментов (GitHub, ChatGPT, VPS, HeyGen, CapCut и т.п.)
- Иностранные термины (HR-скрининг, Reels, Shorts, TikTok, iOS и т.п.)
- Аббревиатуры, непонятные нетехнической аудитории

Формат: `[Термин] — [расшифровка 1-2 предложения простым языком]`

## Строка дизайна (первая строка документа)

Содержит: палитра словами (без hex-кодов), стиль и настроение, что главное на первом экране, референс-сайт (если есть), тематические декоративные элементы.

**Главное — опираться на концепт и тему мероприятия.** Дизайн-советы давать исходя из того, о чём вебинар: подсказывать визуальные образы, релевантные теме.
- Вебинар по презентациям → декоративные слайды/мокапы, инфографика, иконки инструментов
- Вебинар по вайб-кодингу → пиксели, код, терминал
- Вебинар по ИИ-агентам → схемы связей, ноды, «мозг»/нейросеть

**Цвета — только простыми словами**, исходя из темы и ЦА:
- «тёмный фон, электрический синий акцент, белый текст»
- «яркий градиент фиолетовый → малиновый, белый текст»
- «светлый, деловой, тёмно-синий + оранжевый акцент»

НЕ писать hex-коды. Дизайнер сам подберёт точные значения по описанию.

**Не указывать форматы и размеры** (десктоп/мобиль, баннеры TG/VK, размеры в px) — это не задача ТЗ на лендинг.

Доп. элементы (3D-объект, иконки, анимация, образ спикера) — описывать кратко, если есть идея под тему. **Не раздувать** — первый экран не должен быть перегружен. Главный смысл первого экрана берётся из оффера в концепте (он есть почти всегда); декоративные идеи добавлять немного, поверх оффера.

## Форматирование Google Doc

Базовый шрифт — Arial 11pt. Отступы абзацев = 0, пространство только от пустых строк.

| Элемент | Размер | Стиль | Фон / цвет |
|---|---|---|---|
| Строка ДИЗАЙН | 11pt | Обычный | Жёлтый фон `#FFFF00` |
| Разделители `___` | 11pt | Обычный | — |
| Дата (`23 апреля в 13:00мск`) | 11pt | **Жирный** | — |
| Тип мероприятия (`Открытый урок`) | 10pt | Обычный | — |
| Название CAPS (1-2 строки) | 18pt | **Жирный** | — |
| Хук-текст | 11pt | Обычный | — |
| Тире-буллеты `— текст` | 11pt | Обычный | — |
| (CTA кнопки) | 16pt | **Жирный** | — |
| Сноска `*Все иностранные...` | 9pt | Курсив | — |
| `Всем, кто придет на эфир...` | 11pt | **Жирный** | — |
| Список бонусов | 11pt | Обычный | — |
| Заголовки разделов (`ПРОГРАММА:`, `СПИКЕР`, `КОМУ ТОЧНО СТОИТ БЫТЬ?`, `КТО МЫ?`, `ВАРИАНТЫ УЧАСТИЯ`, `ГЛОССАРИЙ ТЕРМИНОВ`) | 14pt | **Жирный** | — |
| Коннектор перед списком программы (`А также поговорим:`, `А также поговорим про:`, `Поговорим про:`, `Обсудим:` и любые аналогичные фразы из концепта, оканчивающиеся на `:`) | 11pt | **Жирный** | — |
| Нумерованные пункты программы | 11pt | Обычный | — |
| Текст спикера (инструкция-ссылка) | 11pt | Обычный | — |
| Заголовки тарифов (`БЕСПЛАТНЫЙ`, `БИЗНЕС`) | 12pt | **Жирный** | — |
| Содержимое тарифов | 11pt | Обычный | — |
| Глоссарий — термин | 11pt | **Жирный** | — |
| Глоссарий — расшифровка | 11pt | Обычный | — |

**Таблица тарифов:** двухколоночная, без видимых границ, колонки равной ширины.

**Пустые строки:**
- 1 пустая строка между блоками
- 0 пустых строк между однотипными элементами подряд (буллеты, бонусы, глоссарий)
- Разделители `___` — отдельная строка с пустой строкой до и после

## Алгоритм работы при получении концепта

1. Прочитать примеры из `фин проект/Сайты для бесплатника/2026-05_лендинги_примеры_ref.md`
2. Извлечь из концепта: дата, время, тип, название, хук, буллеты, бонусы, программа, спикер, ЦА
3. Уточнить недостающее (спикер, дизайн)
4. Собрать все блоки по структуре, составить глоссарий
5. Создать Google Doc с форматированием по таблице
6. Сохранить ссылку в `google-doc-link.txt`
7. Отдать Яне ссылку в чат

## Правила содержания

- **Текст из концепта — дословно**: название, хук, буллеты, программа, бонусы, ЦА — брать как есть, без изменений, переформулировок и сокращений
- Ничего не выдумывать — никаких новых бонусов, пунктов программы, цифр
- CTA-тексты варьировать в рамках одного файла (не повторять одну и ту же кнопку дважды): УЧАСТВОВАТЬ / ПРИСОЕДИНИТЬСЯ / ПРИНЯТЬ УЧАСТИЕ / НЕ ПРОПУЩУ / Я С ВАМИ / ЗАРЕГИСТРИРОВАТЬСЯ / ПОСЕТИТЬ УРОК / ЗАЛЕТАЮ
- **Следить за грамматикой и логикой связок**: коннектор перед списком должен согласовываться с пунктами. Если пункты начинаются с глагола («Разберём…», «Покажем…»), коннектор не должен дублировать действие («А также поговорим:» + «Разберём…» — нелогично, верно «А также:»). Проверять, что заголовок-связка и содержимое читаются как единое предложение.

## Правила безопасности

- Создавать файлы только внутри `фин проект/Сайты для бесплатника/[название]/`
- Не изменять существующие файлы без подтверждения Яны
- НЕ идти на сайты спикеров — только ссылка-инструкция
- Не расшаривать и не публиковать без разрешения
