# 🚀 Скилл «Запуск» — оркестратор подготовки вебинара для Claude Code

Набор Claude Code skills, который из **одного концепта мероприятия** генерирует весь пакет
маркетинговых материалов для запуска вебинара и складывает их в Google Docs: воронку писем
(бот + email), ТЗ на лендинг для дизайнера и ТЗ для копирайтера.

> Проект сделан в рамках работы маркетолога-продюсера онлайн-университета и собран как
> переносимый скилл: клонируешь репозиторий, подключаешь **свой** Google Drive — и оно работает.

---

## Проблема, которую решает

Подготовка одного вебинара — это рутинный конвейер из нескольких документов, которые нужно
собрать быстро, без расхождений и строго по проверенным шаблонам:

- воронка писем для чат-бота и для email (10–20 писем, два согласованных варианта);
- ТЗ на лендинг для дизайнера;
- ТЗ для копирайтера на серию анонсов.

Раньше каждый документ собирался руками по образцам, копировался, переформатировался под
Google Docs (фирменные цвета, жирность CTA, размеры шрифтов). На это уходили часы, а в текстах
накапливались расхождения (кавычки, цифры, эмодзи).

**Скилл «Запуск» сводит это к одному диалогу:** даёшь концепт → отвечаешь на блок вопросов →
получаешь 4 готовых ссылки на Google Docs с корректным форматированием.

## Для кого

Маркетолог / продюсер / project-менеджер, который запускает вебинары и работает в связке
Claude Code + Google Docs. Шаблоны и тексты заточены под русскоязычную EdTech-аудиторию
(AI / no-code / вайб-кодинг).

---

## Как это устроено

`zapusk` — **оркестратор**. Он принимает концепт, задаёт все уточняющие вопросы одним блоком,
создаёт папку на Google Drive и последовательно запускает три субскилла, передавая им `FOLDER_ID`:

```
            ┌─────────────────────────────┐
   концепт →│         zapusk              │→ 4 ссылки на Google Docs
            │       (оркестратор)         │
            └──────────────┬──────────────┘
                           │
        ┌──────────────────┼──────────────────────┐
        ▼                  ▼                       ▼
  voronki-dlya-       tz-dlya-dizov          tz-dlya-kopirov
  zapuskov            (ТЗ на лендинг)        (ТЗ для копирайтера)
  (воронка бот+email)
```

| Скилл | Что делает | Скрипт |
|---|---|---|
| **zapusk** | Оркестрация: вопросы → папка Drive → 3 субскилла → 4 ссылки | `create_drive_folder.py` |
| **voronki-dlya-zapuskov** | Воронка писем: бот-версия + email-версия (2 Google Doc) | `create_voronka_gdoc.py` |
| **tz-dlya-dizov** | ТЗ на лендинг для дизайнера (1 Google Doc) | `create_landing_tz_gdoc.py` |
| **tz-dlya-kopirov** | ТЗ для копирайтера на серию анонсов (1 Google Doc) | `create_kopirov_tz_gdoc.py` |

Python-скрипты в `scripts/` создают и форматируют Google Docs через Google Docs/Drive API:
фирменные цвета, жирность CTA, размеры шрифтов, цветные плашки — всё проставляется автоматически
по правилам из соответствующего `skill.md`.

## Установка

### 1. Клонировать и поставить зависимости

```bash
git clone <URL этого репозитория>
cd <папка репозитория>

python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
```

### 2. Подключить свой Google Drive (обязательно)

> ⚠️ Мои Google-доступы в репозиторий **не входят** (`credentials.json` и `token.json` — личные
> секреты и добавлены в `.gitignore`). Нужно подключить **свой** аккаунт — это 5 минут.

1. Зайди в [Google Cloud Console](https://console.cloud.google.com/) → создай проект.
2. **APIs & Services → Library** → включи **Google Docs API** и **Google Drive API**.
3. **APIs & Services → Credentials → Create credentials → OAuth client ID**
   → тип приложения **Desktop app**.
4. Скачай JSON и положи его как `scripts/credentials.json`
   (шаблон структуры — `scripts/credentials.json.example`).
5. (необязательно) Если хранишь файл в другом месте — скопируй `.env.example` в `.env`
   и пропиши путь в `GOOGLE_CREDENTIALS_PATH`.

При первом запуске любого скрипта откроется браузер для авторизации Google. После входа рядом
появится `scripts/token.json` — он закеширует доступ, больше авторизация не понадобится.
Запрашиваемые права (scopes): создание документов и доступ только к файлам, созданным приложением
(`documents`, `drive.file`).

### 3. Открыть папку в Claude Code

Открой репозиторий в Claude Code — скиллы из `.claude/skills/` подхватятся автоматически.

## Использование

В Claude Code:

```
/zapusk
```

Дальше даёшь концепт мероприятия (тема, дата, спикер, программа, бонусы, ЦА, продаваемый продукт),
отвечаешь на блок вопросов — и получаешь:

```
Папка Drive:   <ссылка на папку>

Воронка бот:   <ссылка>
Воронка email: <ссылка>
Лендинг:       <ссылка>
ТЗ для копира: <ссылка>
```

Скрипты можно запускать и поштучно, например пересоздать только воронку:

```bash
python3 scripts/create_voronka_gdoc.py --file путь/к/файлу.md --title "Название"
# обновить уже созданный документ (без нового URL):
python3 scripts/create_voronka_gdoc.py --file путь/к/файлу.md --doc-id <ID документа>
```

## Безопасность

- `credentials.json`, `token.json`, `.env` — **в `.gitignore`**, в репозиторий не попадают.
- Скрипты запрашивают scope `drive.file` — доступ только к файлам, которые создаёт само
  приложение, а не ко всему Drive.
- Скиллы создают файлы строго внутри своих рабочих папок и не трогают чужие без подтверждения.

## Примечания

- Образцы в `письма-для-воронки/примеры/` и `*_ref.md` — эталоны, по которым скилл пишет новые
  тексты. Это маркетинговые материалы, по ним удобно понять стиль и структуру.
- Тексты и шаблоны на русском и заточены под конкретную EdTech-нишу — при переносе в другую
  компанию правь образцы и инструкции под себя.
