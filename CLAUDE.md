# Контекст для скилла «Запуск»

Этот файл загружается Claude Code как фоновый контекст и помогает скиллам
(`zapusk`, `voronki-dlya-zapuskov`, `tz-dlya-dizov`, `tz-dlya-kopirov`) писать тексты в нужном
тоне и по проверенным формулам. Здесь только то, что нужно для работы скилла — без личных данных.

---

## Ниша и аудитория

Скилл заточен под запуски вебинаров онлайн-университета по AI / no-code / вайб-кодингу /
промпт-инжинирингу. Аудитория русскоязычная, инструменты — доступные в РФ.

**Сегменты аудитории (учитывать при написании любого текста):**
- **Нетехнические**: маркетологи, дизайнеры, предприниматели, контент-мейкеры — хотят
  автоматизировать работу и зарабатывать больше.
- **Технические**: разработчики, промпт-инженеры — хотят освоить вайб-кодинг, API, LLM.
- **Общая боль**: страх отстать от AI-тренда, желание больше зарабатывать, нехватка времени.

При написании всегда учитывать: доступность инструментов в РФ, русскоязычную аудиторию,
IT-образовательную нишу.

## Данные по рассылкам (применять автоматически при работе с текстами)

Из реального анализа email- и Telegram-рассылок.

**Стратегическое правило**: пропорция 4:1 — на 4 контентных письма максимум 1 хайповое.
Никогда два хайповых подряд.

**Лучшие формулы заголовков email:**
- **H1 — Псевдо-уведомление** (OR до 13.4%): «Оформлена заявка 4567🔥»,
  «❗️Регистрация не подтверждена [событие]». Обязательно: первый абзац оправдывает заголовок.
- **H2 — Когнитивный диссонанс** (OR до 11.7%): «💅 Ваша запись на маникюр подтверждена!» —
  для IT-аудитории неожиданно, поэтому работает.
- **K1 — Конкретная задача + снятие барьера** (OR до 13.5%, лучшая формула):
  «Собери бота, который парсит Авито [Своими руками]», «Научу собирать чат-ботов за 30 минут [Бесплатно]».
- **K3 — Узнаваемый бренд + срочность**: «ЗАВТРА! Практикум по AI-экосистеме Google» —
  «ЗАВТРА» работает как реальный дедлайн.
- Всегда есть глагол действия. Скобки — не длиннее 5–7 слов.

**Структура тела email-письма (7 блоков):**
1. Эмпатический хук — боль читателя, НЕ описание программы
2. Кто пишет — 1 предложение, без биографии
3. Первый CTA-кнопка до основного контента
4. Польза по жизненным сценариям (работа / быт / заработок / учёба), не по программе курса
5. Конкретные бонусы с названиями и денежной ценностью
6. Снятие страха («Кодить не нужно!», «Доступно в РФ»)
7. Финальный CTA — эмоционально сильнее первого

**Telegram**: первые 3 строки — про состояние читателя, НЕ про событие. Ссылка после 2–3 абзаца,
не только в конце. Максимум 800–1000 символов. «Двери закрываются» работает лучше,
чем «осталось мало мест».

**Из наблюдений по данным:**
- Хайповые заголовки → OR выше на 30–60%, но CTR ниже в 80% случаев
- Короткие письма работают лучше в 70% случаев
- «Бесплатно» и «ограничение» эффективнее в теле письма, чем в заголовке
- Минимум 1 цифра в письме: «за 30 минут», «$240 бесплатно», «80 000 за проект»

> Полная методичка — в `фин проект/docs/рассылки-инструкция.md` (формулы, крючки, тело письма)
> и `фин проект/тз на письма/финпроект-инструкция.md` (структура ТЗ). Данные лучших рассылок —
> `Доп-но для работы клода/data/рассылки.xlsx`.

## Язык и тон

- Язык — русский.
- Писать **ИИ**, не AI (везде, кроме дословных цитат концепта). Названия инструментов на
  английском оставлять как есть (ChatGPT, GitHub, API и т.п.).
- Пол по возможности обезличивать; если глагол мужского рода — добавлять суффикс для женского.
- Следить за орфографией, пунктуацией и буквой «ё»; вычищать нейроартефакты.
- Тон — по делу, без воды и заискивающих вступлений.

## Правила нейминга файлов

Формула: `[дата]_[тема]_[тип-файла].[расширение]`

Дата — в формате `YYYY-MM`, если она есть. Тип файла (`report`, `doc`, `ref`, `voronka` и т.п.)
помогает быстро понять содержимое. Примеры из этого проекта:
- `2026-05_подработка-на-ии_voronka_bot_doc.md`
- `2026-05_<название>_landing-tz_doc.md`
- `2026-05_<название>_kopirov-tz_doc.md`

Маркеры в имени воронок (`voronka`, `_bot`/`_email`) — обязательны.
